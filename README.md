# 芝しごと・積算温度追跡マップ（AgroMap）

芝生の害虫・病害・雑草の発生リスクを積算温度（GDD）マップで可視化する Web アプリケーションです。  
**v2.0.0** から、植物成長調整剤（プリモマックス・グリーンフィールド）の散布適時判定機能を追加しています。

気温データは [NASA POWER](https://power.larc.nasa.gov/) を利用しています。

---

## 主な機能

### 積算温度マップ（害虫・病害・雑草）

- シバツトガ、スジキリヨトウ、マメコガネ、タマナヤガ、ダラースポット、スズメノカタビラの 6 種
- 前年・当年の週次アニメーションで積算温度の推移を表示
- 左パネルから対象を切り替え、凡例と説明を表示

### 薬剤散布適時判定（v2.0.0）

| 薬剤 | 成分名 | 基準温度 | 判定の目安 |
|------|--------|----------|------------|
| プリモマックス | トリネキサパックエチル | 0℃ | 散布適時 200℃日、リバウンド目安 200℃日超 |
| グリーンフィールド | フルルプリミドール | 0℃ | 散布適時 300～350℃日、リバウンド目安 350℃日超 |

指定した **散布日** から **昨日** までの積算温度を、入力した緯度・経度で計算します。

## 使い方（Web 画面）

ブラウザでアプリを開くと、画面上部に操作パネル、下部にマップが表示されます。

### 1. 場所の指定

1. **緯度・経度** を入力する（初回は東京付近: 35.68, 139.76）。
2. **Enter** を押すと、座標が確定し、マップ上にピンが表示されます。
3. **現在地を取得** を押すと、ブラウザの位置情報から座標を読み込みます（HTTPS 環境を推奨）。
   - 取得に失敗した場合はエラーを表示し、**前回保存した座標** を維持します。

座標は Cookie に **1 年間** 保存され、次回アクセス時に復元されます。

### 2. 薬剤の散布日と積算温度

パネルに次の説明が表示されます。

> ※ 各薬剤の散布日から昨日までの積算温度を表示します

各薬剤ブロックは 2 行構成です。

```
プリモマックス（トリネキサパックエチル）
散布日 [日付] [ゲージ] 判定メッセージ
```

1. **散布日** を選択する（未選択のときは計算しません）。
2. 散布日も Cookie に **1 年間** 保存されます。
3. ゲージ（最大 400℃日）とメッセージが更新されます。

#### プリモマックスのメッセージ

| 積算温度（GDD） | 表示内容 |
|-----------------|----------|
| 200℃日未満 | 現在〇〇GDDです。散布適時まであと〇〇GDDです |
| 200℃日以上 | 現在〇〇GDDです。リバウンドの目安までを〇〇GDD超えています |

#### グリーンフィールドのメッセージ

| 積算温度（GDD） | 表示内容 |
|-----------------|----------|
| 300℃日未満 | 現在〇〇GDDです。散布適時まであと〇〇GDDです |
| 300～350℃日 | 現在〇〇GDDです。散布適時です |
| 350℃日超 | 現在〇〇GDDです。リバウンドの目安までを〇〇GDD超えています |

- GDD は **小数第 1 位** まで表示します。
- ゲージは 200℃日（プリモ）または 300℃日・350℃日（グリーンフィールド）を目安線として表示し、閾値を超えた部分は **赤** で示します。

### 3. 害虫マップの操作

- **左パネル** のボタンで害虫・病害・雑草を切り替えます。
- マップ上の **操作バー** で、前年から当年までのアニメーションを再生・巻き戻し・早送りできます。
- 凡例と説明文は左パネルに表示されます。

## ローカルでの起動

### 必要環境

- Python 3.10 前後
- PostgreSQL（害虫一覧・マップ用の気温 DB。薬剤 GDD は DB がなくても NASA API で計算可能）

### 手順

```powershell
cd c:\Users\hitos\agromap
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python app.py
```

ブラウザで **http://127.0.0.1:5000/** を開きます。

終了はターミナルで `Ctrl+C` です。

### データベース（任意だが推奨）

ローカル既定の接続先:

- ホスト: `localhost`
- ポート: `5432`
- DB 名: `agromap`
- ユーザー / パスワード: `postgres` / `postgres`

本番では環境変数 `DATABASE_URL` を設定してください。

```text
DATABASE_URL=postgresql://user:password@host:5432/agromap
```

初回起動時に `Database` クラスがテーブルと害虫マスタを初期化します。

## API

### 害虫一覧

```http
GET /api/pests
```

### 薬剤 GDD（散布日～昨日）

```http
GET /api/gdd?lat=35.68&lon=139.76&start_date=2026-04-01&base_temp=0
```

| パラメータ | 説明 |
|------------|------|
| `lat` | 緯度 |
| `lon` | 経度 |
| `start_date` | 散布日（`YYYY-MM-DD`） |
| `base_temp` | 基準温度（既定 `0`） |

レスポンス例:

```json
{
  "gdd": 123.4,
  "start_date": "2026-04-01",
  "end_date": "2026-05-28",
  "base_temp": 0,
  "lat": 35.68,
  "lon": 139.76
}
```

DB に近いグリッドの気温があれば DB から、なければ NASA POWER から取得します。

## データ更新（運用）

日次バッチは `fetch_and_update.py` が担当します。

1. NASA POWER から気温データを取得
2. 積算温度を DB に計算・保存
3. 害虫マップ画像を生成
4. アニメーション用フレームを生成

GitHub Actions（`.github/workflows/daily-update.yml`）から毎日実行する構成です。手動実行:

```powershell
python fetch_and_update.py
```

個別スクリプト:

| スクリプト | 用途 |
|------------|------|
| `fetch_temperature_data.py` | 気温取得 |
| `calculate_accumulated_temperature.py` | 積算温度計算 |
| `generate_maps.py` | 静的マップ生成 |
| `generate_animation_data.py` | アニメーションフレーム生成 |

## 注意事項

- 積算温度・判定メッセージは参考情報です。実際の散布判断はラベル表示・気象・芝生の状態などを総合して行ってください。
- 現在地取得はブラウザと OS の位置情報許可が必要です。
- 薬剤 GDD の NASA API 取得は、初回のみ数秒かかることがあります。
